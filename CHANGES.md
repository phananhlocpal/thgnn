## Feature Dimensions (Updated)

**Before:**
- Non-verbal dim: 35 (17 AUs + 18 zeros)

**After:**
- Non-verbal dim: 105 (17 AUs + 88 COVAREP)
- This provides complete multimodal information without dropping COVAREP data

---

## Data Structure (Updated)

**Before (with {ID}_P/ folders):**
```
data/
  {ID}_P/
    {ID}_TRANSCRIPT.csv
    {ID}_CLNF_AUs.txt
    {ID}_covarep.csv
```

**After (flat structure):**
```
data/daicwoz/
  {ID}_TRANSCRIPT.csv
  {ID}_CLNF_AUs.txt
  {ID}_COVAREP.csv
  {ID}_text_feats.npy       ← Generated by extract_bert.py
  train_split_Depression_AVEC2017.csv
  dev_split_Depression_AVEC2017.csv
```

---

## New Requirements

1. **BERT embeddings are now mandatory**: Run `extract_bert.py` before training
2. **COVAREP is now mandatory**: All participants must have `{ID}_COVAREP.csv`
3. **No fallback mechanisms**: All missing data will raise `FileNotFoundError`
4. **Real data only**: No synthetic/demo data whatsoever

---

## Testing Recommendations

1. Test with participant 300 (most complete data)
2. Verify BERT extraction works: `python extract_bert.py --split_csv data/daicwoz/train_split_Depression_AVEC2017.csv`
3. Check feature dimensions match (768-dim text, 105-dim non-verbal)
4. Run training on small batch to verify no errors

---

## Next Steps (For User)

1. Ensure all participant data is in `data/daicwoz/` with correct file naming
2. Run `extract_bert.py` for all participants in your splits
3. Create proper train/dev/test CSV files in `data/daicwoz/`
4. Run `train.py` to train the model
5. Use `inference.py` for evaluation
