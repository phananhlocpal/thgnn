# Setup Guide: DAIC-WOZ TH-GNN Pipeline

## Data Structure

All data files must be in `data/daicwoz/` directory:

```
data/daicwoz/
├── 300_TRANSCRIPT.csv          ← Transcript (TSV format: start_time, stop_time, speaker, value)
├── 300_CLNF_AUs.txt             ← OpenFace Action Units (17 AUs, frame-level)
├── 300_COVAREP.csv              ← Acoustic features (88-dim, frame-level)
├── 300_text_feats.npy           ← BERT embeddings (N_turns, 768) [generated by extract_bert.py]
│
├── 301_TRANSCRIPT.csv
├── 301_CLNF_AUs.txt
├── 301_COVAREP.csv
├── 301_text_feats.npy
│
├── train_split_Depression_AVEC2017.csv
├── dev_split_Depression_AVEC2017.csv
└── test_split_Depression_AVEC2017.csv
```

### CSV Format

**Transcript CSV** (TSV format):
```
start_time	stop_time	speaker	value
36.588	39.668	Ellie	hi i'm ellie thanks for coming in today
62.328	63.178	Participant	good
...
```

**Label CSV** (train/dev/test):
```
Participant_ID,PHQ8_Score,PHQ8_Binary
300,15,1
301,8,0
...
```

## Step-by-Step Setup

### 1. Prepare BERT Text Embeddings

**This must be done FIRST before training.**

```bash
python extract_bert.py \
    --split_csv data/daicwoz/train_split_Depression_AVEC2017.csv \
    --data_root data/daicwoz/
```

This creates `{ID}_text_feats.npy` for each participant.

Optional: Use clinical BERT model for better results:
```bash
python extract_bert.py \
    --split_csv data/daicwoz/train_split_Depression_AVEC2017.csv \
    --data_root data/daicwoz/ \
    --model_name "mental/mental-bert-base-uncased"
```

### 2. Train the Model

```bash
python train.py \
    --train_csv data/daicwoz/train_split_Depression_AVEC2017.csv \
    --dev_csv data/daicwoz/dev_split_Depression_AVEC2017.csv \
    --data_root data/daicwoz/ \
    --epochs 50 \
    --batch_size 8 \
    --hidden_dim 128
```

Output: `checkpoints/best_f1_model.pt` and `checkpoints/best_mae_model.pt`

### 3. Run Inference

```bash
python inference.py \
    --checkpoint checkpoints/best_f1_model.pt \
    --dev_csv data/daicwoz/dev_split_Depression_AVEC2017.csv \
    --data_root data/daicwoz/ \
    --output_csv discrepancy_report.csv
```

## Important Notes

⚠️ **NO SYNTHETIC DATA**: All demo mode has been removed. You must provide real data.

⚠️ **BERT Embeddings Required**: Each participant needs `{ID}_text_feats.npy`. Run `extract_bert.py` first.

⚠️ **File Naming**: Files must follow exact naming: `{ParticipantID}_TRANSCRIPT.csv`, `{ParticipantID}_CLNF_AUs.txt`, etc.

⚠️ **Data Size**: Participant 300 is currently the most complete. Other participants will be added gradually.

---

## Feature Dimensions

- **Text features** (BERT): 768 dimensions (N_turns, 768)
- **Action Units (AUs)**: 17 dimensions (from OpenFace)
- **COVAREP**: 88 dimensions (acoustic features)
- **Non-verbal combined**: 105 dimensions (17 AUs + 88 COVAREP)
- **Model hidden dim**: 128 (default, configurable)

---

## Troubleshooting

### Error: "BERT text features not found"
→ Run `extract_bert.py` for that participant first.

### Error: "Could not load COVAREP"
→ Check that `{ID}_COVAREP.csv` exists and is not corrupted.

### Error: "Participant_ID column not found"
→ Verify your label CSV has column named `Participant_ID` (case-sensitive).
